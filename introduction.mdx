# Welcome to JustBenchThatLLM

Stop guessing. Start measuring. Make data-driven decisions about your AI infrastructure.

**JustBenchThatLLM** is a visual, reproducible benchmarking platform for LLM inference endpoints including vLLM, LM Studio, and OpenAI-compatible APIs. Get production-grade metrics, side-by-side comparisons, and actionable insights to optimize your AI infrastructure.

## Stop wondering, start knowing

JustBenchThatLLM gives you answers, not assumptions:

- **Version upgrades**: Test vLLM updates before they hit production
- **Model performance**: Benchmark Mistral-3B's actual speed on your infrastructure
- **Load capacity**: Discover real limits with stepped load testing
- **Optimization impact**: Measure improvements, don't assume them

## Compare everything

Designed from the ground up for comparative analysis:

- **Models**: Mistral vs Llama vs Qwen on the same infrastructure
- **Configurations**: Test max_tokens, temperature, and concurrency impacts
- **Libraries**: Validate framework updates with confidence
- **Infrastructure**: GPU vs CPU, different memory configs, scaling strategies

## Key capabilities

### Visual intelligence

- **Interactive dashboards**: Filter, zoom, and export your data your way
- **Stepped load analysis**: Automatically find performance limits
- **Timeline view**: Explore request-level details with interactive visualization
- **Radar charts**: Multi-dimensional performance comparison at a glance

### Reproducible workflows

- **Save configurations**: Re-run exact benchmarks with one click
- **Export everything**: CSV and JSON for further analysis
- **Tag and filter**: Organize benchmarks by project, environment, or experiment
- **Historical comparison**: Compare today's deployment with last week's

### Production-ready deployment

- Deploy with Docker Compose in minutes
- Scale with Kubernetes and Helm charts
- Built-in health checks and restart policies
- Dark mode support for extended analysis sessions

## Real-world use cases

### For DevOps and SRE teams

- **Pre-deployment validation**: Test infrastructure changes in staging with production-like loads
- **Capacity planning**: Make data-driven scaling decisions based on actual performance metrics
- **SLO definition**: Set realistic performance targets backed by empirical data

### For ML engineers and data scientists

- **Model selection**: Compare inference speed vs quality trade-offs across different models
- **Configuration tuning**: Find optimal serving parameters for your specific use case
- **Library updates**: Validate new framework releases without production surprises
