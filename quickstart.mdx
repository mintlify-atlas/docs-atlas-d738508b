# Quickstart

Get JustBenchThatLLM up and running in minutes with Docker Compose. This guide will walk you through deployment, initial setup, and running your first benchmark.

## Prerequisites

Before you begin, ensure you have:

- Docker Engine 20.10 or later
- Docker Compose v2.0 or later
- At least 4GB of available RAM
- An LLM inference endpoint to benchmark (vLLM, LM Studio, or OpenAI-compatible API)

<Note>
  If you don't have an inference endpoint yet, you can still deploy JustBenchThatLLM and configure endpoints later.
</Note>

## Deploy with Docker Compose

<Steps>
  <Step title="Clone the repository">
    Clone the JustBenchThatLLM Docker repository to your local machine:

    ```bash
    git clone https://github.com/redteams-fr/JustBenchThatLLM-docker
    cd JustBenchThatLLM-docker
    ```
  </Step>

  <Step title="Start the services">
    Launch all services using Docker Compose. This will pull the required images and start the containers:

    ```bash
    docker-compose up -d
    ```

    The deployment includes:
    - **Redis** (port 6379): Message queue and caching
    - **MongoDB** (port 27017): Metrics and configuration storage
    - **Backend** (port 8000): REST API for benchmark orchestration
    - **Worker**: Benchmark execution engine
    - **Frontend** (port 8005): Web-based dashboard

    <Note>
      The `-d` flag runs containers in detached mode. Services will restart automatically unless stopped.
    </Note>
  </Step>

  <Step title="Verify the deployment">
    Check that all services are running and healthy:

    ```bash
    docker-compose ps
    ```

    You should see all five services with status `Up` or `Up (healthy)`.

    To view service logs:

    ```bash
    # All services
    docker-compose logs -f

    # Specific service
    docker-compose logs -f backend
    ```
  </Step>

  <Step title="Access the web interface">
    Open your browser and navigate to:

    ```
    http://localhost:8005
    ```

    You should see the JustBenchThatLLM dashboard with an empty state prompting you to configure your first endpoint.
  </Step>
</Steps>

## Architecture overview

The Docker Compose deployment consists of the following components:

<CodeGroup>
```yaml docker-compose.yml
services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

  mongo:
    image: mongo:8.0
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db

  backend:
    image: redteamsfr/benchmark-backend:latest
    ports:
      - "8000:8000"
    environment:
      - REDIS_URL=redis://redis:6379/0
      - MONGODB_URL=mongodb://mongodb:27017
      - STORAGE_TYPE=mongodb

  worker:
    image: redteamsfr/benchmark-worker:latest
    environment:
      - REDIS_URL=redis://redis:6379/0
      - MONGODB_URL=mongodb://mongodb:27017

  frontend:
    image: redteamsfr/benchmark-frontend:latest
    ports:
      - "8005:8005"
    environment:
      - BACKEND_BASE_URL=http://backend:8000
```
</CodeGroup>

### Component responsibilities

- **Redis**: Manages the task queue for benchmark jobs and provides fast caching
- **MongoDB**: Stores benchmark configurations, results, and historical data
- **Backend**: REST API that orchestrates benchmarks and serves dashboard data
- **Worker**: Executes benchmarks by sending requests to your LLM endpoints and collecting metrics
- **Frontend**: Web interface for configuration and visualization

## Troubleshooting

### Services won't start

If services fail to start, check for port conflicts:

```bash
# Check if ports are already in use
lsof -i :8005  # Frontend
lsof -i :8000  # Backend
lsof -i :6379  # Redis
lsof -i :27017 # MongoDB
```

You can modify ports in `docker-compose.yml` if needed.

### Can't access the UI

Verify the frontend container is running:

```bash
docker-compose logs frontend
```

Ensure the backend is accessible:

```bash
curl http://localhost:8000/health
```

### Reset everything

To stop services and remove all data:

```bash
# Stop and remove containers
docker-compose down

# Remove volumes (WARNING: deletes all benchmarks and configurations)
docker-compose down -v
```
