## Overview

JustBenchThatLLM can be deployed using Docker Compose for local development and testing. The stack includes Redis for task queueing, MongoDB for data persistence, a FastAPI backend, a worker service, and a Streamlit frontend.

## Complete docker-compose.yml

```yaml
services:
  redis:
    image: redis:7-alpine
    container_name: redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    command: ["redis-server", "--appendonly", "yes"]
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  mongo:
    image: mongo:8.0
    container_name: mongodb
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
      interval: 5s
      timeout: 3s
      retries: 5

  backend:
    image: redteamsfr/benchmark-backend:latest
    container_name: benchmark-backend
    ports:
      - "8000:8000"
    environment:
      - REDIS_URL=redis://redis:6379/0
      - LOG_LEVEL=INFO
      - STORAGE_TYPE=mongodb
      - MONGODB_URL=mongodb://mongodb:27017
      - MONGODB_DB_NAME=benchmark_db
    depends_on:
      redis:
        condition: service_healthy
      mongo:
        condition: service_healthy
      worker:
        condition: service_started
    restart: unless-stopped

  worker:
    image: redteamsfr/benchmark-worker:latest
    container_name: benchmark-worker
    environment:
      - REDIS_URL=redis://redis:6379/0
      - LOG_LEVEL=INFO
      - MONGODB_URL=mongodb://mongodb:27017
      - MONGODB_DB_NAME=benchmark_db
      - STORAGE_TYPE=mongodb
    depends_on:
      redis:
        condition: service_healthy
      mongo:
        condition: service_healthy
    restart: unless-stopped

  frontend:
    image: redteamsfr/benchmark-frontend:latest
    container_name: benchmark-frontend
    ports:
      - "8005:8005"
    environment:
      - BACKEND_BASE_URL=http://backend:8000
    restart: unless-stopped

volumes:
  redis_data:
  mongo_data:
```

## Services

### Redis

<ParamField path="image" type="string" default="redis:7-alpine">
  Redis image for task queue management
</ParamField>

<ParamField path="ports" type="array" default="6379:6379">
  Exposes Redis on port 6379
</ParamField>

<ParamField path="volumes" type="array">
  Persistent storage at `/data` using named volume `redis_data`
</ParamField>

<ParamField path="command" type="array">
  Runs Redis with append-only file (AOF) persistence enabled
</ParamField>

<ParamField path="healthcheck" type="object">
  **test**: `redis-cli ping`
  
  **interval**: 5s
  
  **timeout**: 3s
  
  **retries**: 5
</ParamField>

### MongoDB

<ParamField path="image" type="string" default="mongo:8.0">
  MongoDB 8.0 image for benchmark data storage
</ParamField>

<ParamField path="ports" type="array" default="27017:27017">
  Exposes MongoDB on port 27017
</ParamField>

<ParamField path="volumes" type="array">
  Persistent storage at `/data/db` using named volume `mongo_data`
</ParamField>

<ParamField path="healthcheck" type="object">
  **test**: `mongosh --eval "db.adminCommand('ping')"`
  
  **interval**: 5s
  
  **timeout**: 3s
  
  **retries**: 5
</ParamField>

### Backend

<ParamField path="image" type="string" default="redteamsfr/benchmark-backend:latest">

<ParamField path="depends_on" type="object">

## Usage

```bash
# Start all services
docker compose up -d

# View logs
docker compose logs -f

# Stop all services
docker compose down

# Stop and remove volumes
docker compose down -v
```

## Customization

You can override default environment variables by creating a `.env` file:

```bash
# .env
LOG_LEVEL=DEBUG
MONGODB_DB_NAME=my_benchmark_db
```

Or by modifying the `environment` sections in the docker-compose.yml file directly.
